# Tech

## Nature technique du projet

Il n'y a pas de stack logiciel. Les aspects techniques concernent les formats de fichiers, les outils d'ecriture, les normes academiques et la methode de gestion documentaire.

## Formats presents

- HTML exporte depuis Google Docs ou documents convertis dans `refs/`.
- PDF, DOCX et PPTX originaux dans `refs-raw/`.
- Markdown pour la documentation Obsidian dans `documentation/`.

## Regles academiques structurantes

- Langue de redaction : francais.
- Normes de references : APA 7, adaptation francaise/europeenne selon le guide BIRSG/HERS.
- Sources attendues : sources professionnelles et scientifiques ; eviter vulgarisation, blogs et forums.
- Citations directes : localisation obligatoire.
- Paraphrases : reference obligatoire.
- Liste de references : uniquement les sources citees ou paraphrasees.

## Regles de presentation BIRSG/HERS

- Format A4, orientation portrait sauf besoin specifique.
- Marges de 2,5 cm ; 3 cm a gauche si le travail est relie.
- Police Times New Roman 12 ou Arial 11.
- Interligne 1,5 pour le texte.
- Texte justifie.
- Pagination en chiffres arabes, page de titre comptee mais non numerotee.
- Titres courts, precis, non soulignes, sans point final.

## Methode d'analyse conceptuelle

Pour chaque concept principal, developper :

- definition ;
- attributs ;
- antecedents, prerequis, causes ou facteurs de risque ;
- consequences, benefices ou resultats ;
- commentaire avec les mots de l'etudiante ;
- retour aux constats ;
- lien logique avec la partie suivante.

## Outils de documentation

La documentation est structuree pour Obsidian. Les liens internes doivent utiliser la syntaxe `[[Nom du document]]` lorsque des fichiers du dossier `documentation/` sont cites entre eux.
